# Ancient Game Server — Документация разработки

## 📌 Общее описание

Серверная часть игры «Древные» — стратегии в жанре MMO с изометрическим видом, гексагональной картой и фокусом на управлении героями.  
Проект написан на **ASP.NET Core 8.0**, использует **in-memory** хранение (планируется переход на PostgreSQL).

**API стиль:** REST  
**Порт по умолчанию:** `http://localhost:5200`  
**Swagger:** `http://localhost:5200/swagger`

### 2. Генерация мира

- [x] **5 континентов**  
  Четыре стартовых + центральный (Атлантида, открывается позже).  
  Каждый континент имеет уникальные бонусы.

- [x] **Города (соты)**  
  На каждом континенте генерируется **120 городов**, расположенных концентрическими кругами:  
  - Уровень 1 (внешний круг) — 36 городов  
  - Уровень 2 — 30 городов  
  - Уровень 3 — 24 города  
  - Уровень 4 — 18 городов  
  - Уровень 5 (центр) — 12 городов  

  Координаты городов вычисляются по формуле:  
  `centerX = RingRadius * cos(angle)`  
  `centerZ = RingRadius * sin(angle)`

- [x] **Внутренняя структура города**  
  Каждый город содержит:
  - Замок (центр, координаты 0.5,0.5 в локальных координатах)
  - **10 шахт** (платина / энергия духа)
  - **20 мобов** (4 типа: воители, чародеи, целители, охотники)
  - **3 подземелья** (обновляются раз в сутки)

- [x] **Проверка наложений**  
  При генерации объектов проверяется, что они:
  - не пересекаются друг с другом (minDistance = 0.1)
  - не находятся в центре города (радиус 0.15)

- [x] **Динамические поместья**  
  Поместья игроков **не генерируются заранее**, а создаются при спавне игрока в свободном месте с проверкой наложения на шахты, мобов и подземелья.

### 3. API эндпоинты (реализованные)

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/game/ping` | Проверка работы сервера |
| GET | `/api/game/test` | Тестовый эндпоинт |

| GET | `/api/game/cities` | Список городов на континенте (фильтр по координатам) |

### 4. Логика спавна игрока

- [x] **Выбор города**  
  Запрос должен содержать `continentId` и `cityId`.

- [x] **Проверки**  
  - Город существует и принадлежит континенту.
  - Город имеет уровень **1** (внешний круг).
  - В городе менее 40 поместий.

- [x] **Генерация поместья**  
  - Случайные координаты в пределах города.
  - Проверка наложения с шахтами, мобами, подземельями и другими поместьями.
  - Координаты сохраняются в `Player.SettlementLocalX/Z`.

- [x] **Мировые координаты**  
  `worldX = city.CenterX + (localX - 0.5f) * 20f`  
  `worldZ = city.CenterZ + (localZ - 0.5f) * 20f`

- [x] **Ответ сервера**  
  Возвращает `cityId`, `cityLevel`, `settlementLocal`, `spawnPosition`.

### 5. Вспомогательные методы

- [x] `UpdateContinentCounters()` — пересчёт количества игроков на континентах.
- [x] `IsTooCloseToCenter()` — проверка близости к центру города.
- [x] `IsOverlappingWithAll()` — проверка пересечения со всеми объектами в городе.
- [x] `RespawnMine()` — заглушка для будущего респавна шахт.
